# 要件定義書

## 1. 概要・目的 🎯
- 本プロジェクトは「コンテンツ（記事/ケーススタディ）＋お問い合わせ/ニュースレター」機能を有する情報発信サイトを構築することを目的とします。
- 既存の Headless WordPress（GraphQL）をコンテンツ管理に利用し、Next.js（App Router）でフロントエンドを実装、Prisma で受信データを保持します。

## 2. ゴール
- サイトが安定してデプロイされ、問い合わせ・ニュースレター登録が正しく動作すること。
- テストと CI による品質保証が確立されていること。

## 3. 対象機能
- フロント：Home, Posts, Post detail, Category, Tag, Search, Contact, Newsletter
- API：POST /api/contact, POST /api/newsletter, GET /api/test-db
- 管理：WordPress（コンテンツ）、Prisma（受信データ）

## 4. 非機能要件
- 型安全（TypeScript）・ユニットテスト（Vitest）・Lint（ESLint）を必須とする
- デプロイは Vercel を想定
- セキュリティ：機密情報は環境変数で管理

## 5. 受け入れ基準
- `npm run build` が成功すること
- Contact と Newsletter API が DB 保存と適切なステータスを返すこと
- CI（Lint/Typecheck/Tests）が PR 時に必須で通ること
