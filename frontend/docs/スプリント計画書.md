# スプリント計画書（MVP）

## 目的
- 学習ハブ機能を含む MVP を 2 週間で実装し、基本機能と CI を整備する。

## 期間
- 2 週間（2 スプリント × 1 週間または 1 スプリント × 2 週間に分割可）

## スプリント 1（Week 1） — 基盤整備
- Day 1: 要件確認・設計（要件定義書確定、データモデル設計）
- Day 2–3: Prisma モデル追加（Newsletter, Contact, User） + マイグレーション
- Day 4–5: Next.js の基本ページ scaffold（Home, Posts, Post detail）
- Day 5: CI 基盤（Lint/Typecheck/Tests）を整備

## スプリント 2（Week 2） — 機能実装と仕上げ
- Day 6: Contact / Newsletter API 実装 + Prisma 保存
- Day 7: Course 構造（簡易）と CourseList / CourseCard 実装
- Day 8: テスト追加（API と主要コンポーネント）
- Day 9: E2E（オプション） + ドキュメント整備
- Day 10: バッファ / リリース準備 / デプロイ（Vercel）

## リソース・担当
- 開発者: 1–2 人（フロント/バック兼任）
- レビュー: PR ベースでチーム内レビュー

## 成果物・受け入れ基準
- PR の CI が全て通過していること
- Contact と Newsletter が本番 DB にデータを保存すること
- 主要ページが表示され、WP からコンテンツ取得ができること

## リスク管理
- WP スキーマ差分 → 事前検証テストとフェールセーフ実装
- メール送信の失敗 → リトライ / ログ記録 / 管理通知
